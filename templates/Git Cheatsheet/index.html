Welcome, This Git CheatSheet is for You!

Command Description
git init The first step is to initialize a new Git repo locally in your project root. You can do so with the command below
git add . To add all files in your project to the staging area
git status Shows the status of the current repository including staged, unstaged, and untracked files
git commit -m "your commit message here" A commit message is made up of a short summary of changes. This command is used to commit your changes
git commit -a -m"your commit message here" You can add and commit tracked files with a single command by using the -a and -m options
git log This command shows the commit history for the current repository
git branch You can view all created branches using the git branch command. Highlights current branch with an asterisk
git checkout -b branch_name In a single command, you can create and switch to a new branch right away
git branch -d branch_name When you are done working with a branch and have merged it, you can delete it using this command
git merge branch_name To merge the history of the branch you are currently in with the branch_name
git push When all your work is ready to be saved on a remote repository, you can push all changes using the command below
git pull If other team members are working on your repository, you can retrieve the latest changes made to the remote repository with the command below
git push -u origin branch_name If you want to push a branch to a remote repository you can use the command below. Just remember to add -u to create the branch upstream
